# DeciFlow 测试计划

## 📋 测试概述

本测试计划涵盖了 DeciFlow AI 数据分析工具的完整测试体系，确保产品质量和用户体验。

## 🎯 测试目标

1. **功能完整性** - 验证所有核心功能正常工作
2. **数据准确性** - 确保 AI 查询和数据展示的准确性
3. **安全性** - 防止 SQL 注入和数据泄露
4. **性能表现** - 保证响应速度和系统稳定性
5. **用户体验** - 确保界面交互流畅自然

## 🧪 测试类型

### 1. AI 功能测试
- **自然语言转 SQL**：验证 AI 将中文查询转换为 SQL 的准确性
- **智能分析**：测试数据分析建议的相关性
- **上下文记忆**：验证多轮对话的上下文保持
- **性能**：测试 AI 响应速度

**测试文件**: `src/test/ai-functionality.test.ts`

### 2. 数据库连接测试
- **多数据源支持**：PostgreSQL、MySQL、MongoDB 连接
- **查询执行**：验证 SQL 查询的正确性
- **连接管理**：测试连接池和超时处理
- **错误处理**：验证异常情况的优雅处理

**测试文件**: `src/test/database-connection.test.ts`

### 3. 图表可视化测试
- **图表类型选择**：自动推荐合适的图表类型
- **渲染性能**：测试大量数据的渲染速度
- **交互功能**：缩放、悬停、筛选等交互
- **响应式设计**：适应不同屏幕尺寸

**测试文件**: `src/test/chart-visualization.test.ts`

### 4. 数据字典测试
- **指标管理**：添加、更新、删除业务指标
- **字段定义**：维护数据字段的标准定义
- **智能搜索**：快速查找相关数据元素
- **团队协作**：支持多用户协作编辑

**测试文件**: `src/test/data-dictionary.test.ts`

### 5. 安全功能测试
- **SQL 注入防护**：验证危险 SQL 的拦截
- **数据脱敏**：测试敏感数据的自动脱敏
- **访问控制**：基于角色的权限管理
- **审计日志**：完整的操作记录和追踪

**测试文件**: `src/test/security-functionality.test.ts`

## 📊 测试策略

### 单元测试 (Unit Tests)
- 使用 Vitest 框架
- 覆盖所有核心功能和工具函数
- 模拟外部依赖，确保测试独立性

### 集成测试 (Integration Tests)
- 测试各模块之间的交互
- 验证数据库连接和数据流
- 测试 API 和 UI 的集成

### 性能测试 (Performance Tests)
- 查询响应时间测试
- 大数据量处理能力
- 并发用户访问模拟

### 安全测试 (Security Tests)
- SQL 注入攻击模拟
- 敏感数据泄露检测
- 权限越界尝试

## 🚀 运行测试

### 基本测试
```bash
npm run test
```

### 监视模式
```bash
npm run test:watch
```

### 覆盖率报告
```bash
npm run test:coverage
```

### 运行特定测试文件
```bash
vitest run src/test/ai-functionality.test.ts
```
